<memberOf>

<memberOf> specifies class membership of the documented element or class. [22.4.3 Classification of Components]
Moduletagdocs — Documentation Elements
Attributesatt.global (@xml:id, @n, @xml:lang, @xml:base, @xml:space) (att.global.rendition (@rend, @style, @rendition)) (att.global.linking (@corresp, @synch, @sameAs, @copyOf, @next, @prev, @exclude, @select)) (att.global.analytic (@ana)) (att.global.facs (@facs)) (att.global.change (@change)) (att.global.responsibility (@cert, @resp)) (att.global.source (@source))
key key¶specifies the identifier for a class of which the documented element or class is a member or subclass
Status Optional
Datatype teidata.name
mode mode¶specifies the effect of this declaration on its parent module.
Status Optional
Datatype teidata.enumerated
Legal values are:
add
this declaration is added to the current definitions [Default]
delete
this declaration and all of its children are removed from the current setup
max max¶supplies the maximum number of times the element can occur in elements which use this model class in their content model
Status Optional
Datatype teidata.numeric
min min¶supplies the minumum number of times the element must occur in elements which use this model class in their content model
Status Optional
Datatype teidata.numeric

character data
Note

Elements or classes which are members of multiple (unrelated) classes will have more than one memberOf element, grouped by a classes element. If an element is a member of a class C1, which is itself a subclass of a class C2, there is no need to state this, other than in the documentation for class C1.

Any additional comment or explanation of the class membership may be provided as content for this element.

Example
<memberOf key="model.divLike"/>
<memberOf key="att.identified"/>
