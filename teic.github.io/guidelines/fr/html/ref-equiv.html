<equiv>

<equiv> (équivalent) précise un composant considéré comme un équivalent de l'élément parent, soit par une référence commune, soit par un lien externe. [3.3.4 Terms, Glosses, Equivalents, and Descriptions 22.4.1 Description of Components]

Note

L'attribut mimeType doit être utilisé pour fournir le type de media MIME du script de filtre spécifié par l'attribut filter.

Exemple

L'exemple suivant déclare que l'élément <bo> est conceptuellement équivalent au marqueur construit <hi rend='bold'>, et qu'une définition externe de ce concept est disponible à l'URI indiqué.

<elementSpec ident="hi" mode="change">
 <equiv name="BOLD"/>
 <desc>caractères gras</desc>
 <attList>
  <attDef ident="rend" mode="change">
   <valList>
    <valItem ident="bold"/>
   </valList>
  </attDef>
 </attList>
</elementSpec>
<elementSpec ident="bo" mode="add">
 <equiv name="BOLD"
  uri="http://www.example.com/typesetting/bold"/>
</elementSpec>
