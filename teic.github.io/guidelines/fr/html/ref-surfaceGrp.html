<surfaceGrp>

<surfaceGrp> defines any kind of useful grouping of written surfaces, for example the recto and verso of a single leaf, which the encoder wishes to treat as a single unit. [11.1 Digital Facsimiles]
Moduletranscr — Representation of Primary Sources
Attributsatt.global (@xml:id, @n, @xml:lang, @xml:base, @xml:space) (att.global.rendition (@rend, @style, @rendition)) (att.global.linking (@corresp, @synch, @sameAs, @copyOf, @next, @prev, @exclude, @select)) (att.global.analytic (@ana)) (att.global.facs (@facs)) (att.global.change (@change)) (att.global.responsibility (@cert, @resp)) (att.global.source (@source)) att.declaring (@decls) att.typed (@type, @subtype)

Note

Where it is useful or meaningful to do so, any grouping of multiple surface elements may be indicated using the surfaceGrp elements.

Exemple
<sourceDoc>
 <surfaceGrp>
  <surface ulx="0" uly="0" lrx="200"
   lry="300">
   <graphic url="Bovelles-49r.png"/>
  </surface>
  <surface ulx="0" uly="0" lrx="200"
   lry="300">
   <graphic url="Bovelles-49v.png"/>
  </surface>
 </surfaceGrp>
</sourceDoc>
